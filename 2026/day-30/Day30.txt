### Task 3: Container Lifecycle
Practice the full lifecycle on one container:
1. **Create** a container (without starting it)
docker create container_name ----> it will going to create the container but it will not run it
2. **Start** the container
docker start container ----> it will run the container
3. **Pause** it and check status
docker pause container ----> pause the container
4. **Unpause** it
docker unpause container ---> back to running
5. **Stop** it
docker stop container ---> stop the container gracefullly
6. **Restart** it
docker restart container ---> restart the container
7. **Kill** it
docker kill container ---> forcefully stop the container
8. **Remove** it
docker rm container ---> remove the container



### Task 4: Working with Running Containers
1. Run an Nginx container in detached mode
docker run -d -p 80:80 nginx
2. View its **logs**
docker logs <container_id>

3. View **real-time logs** (follow mode)
docker logs -f <container_id> ---> f means follw mode here
4. **Exec** into the container and look around the filesystem
docker exec -it mynginx bash
ls 
pwd
cd /

5. Run a single command inside the container without entering it
docker exec <container_id> <command>
docker exec 4d7e37aca498 pwd
6. **Inspect** the container — find its IP address, port mappings, and mounts
docker inspect 4d7e37aca498




### Task 5: Cleanup
1. Stop all running containers in one command
docker stop $()
2. Remove all stopped containers in one command
docker container prune 
3. Remove unused images
docker image prune
4. Check how much disk space Docker is using
docker system df
