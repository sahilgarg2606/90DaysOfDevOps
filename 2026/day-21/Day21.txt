String comparisons — =, !=, -z, -n
= ---> check if two string are equal
!= --> string not equal
-z ---> means string is empty
-n ---> means string is not empty

Integer comparisons — -eq, -ne, -lt, -gt, -le, -ge 
-eq means equal to 
-ne means not equal
-lt means less then 
-gt means greater then
-le means less then equal to
-ge means greater then equal to

File test operators — -f, -d, -e, -r, -w, -x, -s
-f ---> check for file exist 
-d ---> check for directory exist or not
-e ---> file exists of any type
-r ---> Read permissions
-w ---> write permission
-x ---> executable permission
-s ---> file not empty

if, elif, else syntax 

if <condition>; then
    echo "condition true"
fi
if <condition>; then
    echo "condition true"
elif <condition2>:then
    echo "condition 2 solve"
else 
    echo "default part if 1 and 2 conditions fail"
fi
Logical operators — &&, ||, !
&& ---> Run second command only if first command succeeds
|| ---> Run second command only if first command fails (non-zero exit code)
! ---> Reverses condition.

Case statements — case ... esac

#!/bin/bash

read -p "Enter environment: " env

case "$env" in
    dev)
        echo "Development environment"
        ;;
    prod)
        echo "Production environment"
        ;;
    *)
        echo "Invalid environment"
        ;;
esac

Task 5: Text Processing Commands
Document the most useful flags/patterns for each:

grep — search patterns, -i, -r, -c, -n, -v, -E

grep = global regular expression print
grep "root" /etc/passwd
shows lines containing root
grep -i "error" file.Text ----->Finds "ERROR", "Error", "error"
it search for case sensitive

grep -r
r means recursive search 
grep -r "database" /var/www/  -----> Searches inside directory and subdirectories

grep -c  ---> count matches
grep -c "failed" log.txt ----> Shows number of matching lines

grep -n ---> show line Numbers
grep -n "nginx" log.txt  ---> Display matching line Numbers

grep -v ----> Show invert matches
grep -v "INFO" app.log ----> shows the lines that donot contain INFO


awk — print columns, field separator, patterns, BEGIN/END
awk process text line by line and splits each line into fields (columns)
awk {' print $1'} file.txt  ---- print single columns 
awk {' print $1 , $3'} file.txt  ---- print multiple columns 

field separator
By default → space
To change separator, use -F
awk -F ':' '{print $1, $7}' /etc/passwd

patterns --> you can filter lines before printing
awk '$1 == "root"' file.txt

BEGIN and END Blocks

Used to execute code:
Before file processing → BEGIN
After file processing → END
awk '
BEGIN { print "Start Processing" }
{ print $1 }
END { print "Done" }
' file.txt

sed is Stream editor editor the file line by line without opening the file
