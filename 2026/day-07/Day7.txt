Linux File System Hierarchy
/ : starting point of everthing
/home: contain user and user folders like Ubuntu
/root: root adminstrative files 
/etc: coonfiguration files 
/var/log : contains log files system and application logs syslog and auth.log
/tmp: toring temporary data during scripts
/bin : system command binaries like cp mv
/sbin : system related binaries like reboot
/usr/bin : contains user install binaries and application like docker , nginx
/opt : used for manually installed apps

Scenario 3: Finding Service Logs

A developer asks: "Where are the logs for the 'docker' service?"
The service is managed by systemd.
What commands would you use?

# Check service status first
systemctl status ssh

# View last 50 lines of logs
journalctl -u ssh -n 50

# Follow logs in real-time
journalctl -u ssh -f
