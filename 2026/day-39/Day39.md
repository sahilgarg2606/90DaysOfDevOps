### Task 3: Pipeline Anatomy
A pipeline has these parts — write what each one does:
- **Trigger** — what starts the pipeline
trigger ---> this is event that start the pipeline and it defines when the pipeline should run like developer pushes the code , manual trigger and pull request
- **Stage** — a logical phase (build, test, deploy)
a stage is the logical phrasse of pipeline
build test and deploy each will runs sequentially
- **Job** — a unit of work inside a stage
a job is the set of tasks executed inside a stage jobs can be run parrallelsm and seqentially
- **Step** — a single command or action inside a job
a step is the single command or action that is run inside the job
npm install
dotnet build
- **Runner** — the machine that executes the job
A runner is the server/machine that executed the jobs
every job needs a runner
self hosted and cloud runner (github)
- **Artifact** — output produced by a job
Artifacts are the outputs generated by jobs and passed to other stages or stored.
