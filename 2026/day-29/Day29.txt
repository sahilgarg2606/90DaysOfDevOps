### Task 3: Run Real Containers

1. Run an **Nginx** container and access it in your browser
docker run -d -p 80:80 nginx
docker run will be run the container and -d means it will run the container in the detach mode
-p means exposing port left side number indicates the host port and right side of number represents the container port and then there is name of the image 
access it from the browser and in the browser just type the localhost:80
2. Run an **Ubuntu** container in interactive mode — explore it like a mini Linux machine
docker run -d Ubuntu
docker exec -it <container_id> bash
now you will able to run the liunx
3. List all running containers
docker ps ---- this command will list all the running containers
4. List all containers (including stopped ones)
--->docker ps -a ---> this command will list the container even it was stopped
5. Stop and remove a container
docker stop <container_id> ----> stop the container 
docker rm <container_id> removing the container



### Task 4: Explore
1. Run a container in **detached mode** — what's different?
docker run -d ubuntu
with d service runs in the background and terminal will be free
2. Give a container a custom **name**
--name=<name>
name tag is used to give the container names 
docker run --name=sahil hello-world
3. Map a **port** from the container to your host
docker run -d -p 80:80 nginx
left side number indicates the host port and right side of number represents the container port 
4. Check **logs** of a running container
docker logs <container_id>
5. Run a command **inside** a running container
docker exec -it ubuntu bash
